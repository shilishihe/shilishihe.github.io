Dio + 阿里云 HTTPDNS（HTTP API）实现跨平台动态域名解析直连方案

纯 Dart 实现，支持 Android / iOS / Windows，零平台通道、零 SDK 依赖。

1. 背景与目标

需求 说明
跨平台 Android / iOS / Windows 三端共用一份代码
不引入官方 SDK 仅使用阿里云 HTTPDNS 的 HTTP API
网络库 Flutter / Dart 的 Dio
最终效果 请求时自动把域名解析成 IP，再以 IP 直连，解决 DNS 劫持、时延等问题

2. 阿里云 HTTPDNS HTTP API 术语介绍

字段 释义
启动IP 调度服务的IP（可能包含域名）入口，用于获得最新的服务IP地址列表。
服务IP 解析服务的IP入口，用于调用域名解析服务。
解析服务 即HTTPDNS解析服务接口，用于将域名解析为对应IP地址。
调度服务 即本文档介绍的提供调度接口的服务，它根据请求来源IP，返回服务IP地址列表及其他服务调度信息，您通常只需要关注返回的IP地址列表。

8. 平台差异注意点

平台 注意
Android 9+ 默认禁止明文 HTTP，需在 network_security_config.xml 允许 203.107.1.33，或改用 HTTPS 解析节点
iOS ATS 同样限制，建议 HTTPS 节点
Windows 无额外限制

9. 性能与安全

  • 并发：单节点 200 QPS 足够日常 App，如流量大可用多节点 + 本地缓存。
  • HTTPS：通过 serverName 保留原域名，证书校验正常。
  • 安全：解析节点走 HTTPS 可避免中间人篡改解析结果。

10. 常见问题 FAQ

现象 原因 解决
证书验证失败 未设置 serverName 示例代码已处理
连接超时 本地网络禁止 203.107.1.33 换 HTTPS 节点或配置代理
返回空字符串 域名未在阿里云备案 控制台添加域名
